## Trigger
**You see in the problem:** "connected components", "number of islands", "reachability", "tree traversal (pre/in/post-order)", "cycle detection", "path that satisfies condition X".

**Reach for this pattern when:**
- You need to visit **every node** of a graph/tree, ordering doesn't matter (or post-order does).
- Counting **connected components** / flood fill.
- Detecting **cycles** in directed/undirected graphs (color/state-set DFS).
- The problem has a **recursive natural structure** — trees, nested structures, regions.

**Don't reach for it when:**
- You need *shortest* path in unweighted graph → BFS.
- The graph is huge and you'll blow the recursion stack — switch to iterative DFS or BFS.
- You need every path with backtracking choices — use backtracking (DFS's stricter cousin).

## The mental model
Go deep along one branch until you can't, then backtrack. Either recursion (implicit stack) or an explicit `stack`. Mark visited so you don't loop. For trees, the call structure mirrors the tree; for graphs, you also need `visited`. Three colors (`WHITE/GRAY/BLACK`) detect cycles in directed graphs.

## Skeleton

```python
# Recursive DFS on a graph
def dfs(node, graph, seen):
    if node in seen: return
    seen.add(node)
    for nxt in graph[node]:
        dfs(nxt, graph, seen)

# Iterative DFS (when stack depth is a worry)
def dfs_iter(start, graph):
    stack, seen = [start], set()
    while stack:
        node = stack.pop()
        if node in seen: continue
        seen.add(node)
        for nxt in graph[node]:
            if nxt not in seen: stack.append(nxt)

# Tree traversals (recursive)
def inorder(node, out):
    if not node: return
    inorder(node.left, out)
    out.append(node.val)
    inorder(node.right, out)

# Cycle detection in directed graph (3-color)
WHITE, GRAY, BLACK = 0, 1, 2
def has_cycle(node, graph, color):
    color[node] = GRAY
    for nxt in graph[node]:
        if color[nxt] == GRAY: return True
        if color[nxt] == WHITE and has_cycle(nxt, graph, color): return True
    color[node] = BLACK
    return False
```

## Complexity
- Time: **O(V + E)** for graphs; **O(n)** for trees.
- Space: **O(V)** for visited + recursion stack (worst case O(V) on a path).

## Variants in this pattern
1. **Graph traversal** — visit all nodes, count components.
2. **Grid flood fill** — connected regions, islands, color regions.
3. **Tree traversals** — pre / in / post-order; iterative versions with explicit stack.
4. **Path sum / path tracking** — DFS that carries running state (sum, path list).
5. **Cycle detection** — 3-color for directed; parent-check for undirected.
6. **Iterative DFS for deep graphs** — when recursion limit is a problem.

## Common bugs / pitfalls
- **Forgetting `visited`** → infinite recursion on cyclic graphs.
- **Marking visited at wrong time** — for grid flood fill, mark when *entering*, not after returning, or you re-enter on the parallel branch.
- **Recursion limit**: Python defaults to 1000. Use `sys.setrecursionlimit(10**6)` or go iterative for big graphs.
- **Undirected cycle detection** can't use 3-color alone — you must skip the parent edge.
- **Mutating the input grid as visited** is clever (no extra space) but make sure to restore if the caller cares.
- **Pre/in/post-order confusion**: in-order on a BST gives sorted output; pre-order serializes; post-order = "children first" (deletion, eval).

## In 30 seconds
Go deep, mark visited, backtrack. If shortest path doesn't matter and the structure is recursive — DFS. For directed cycles, paint nodes WHITE/GRAY/BLACK.
