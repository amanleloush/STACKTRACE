## Trigger
**You see in the problem:** "all subsets / all permutations / all combinations", "N-queens", "Sudoku", "generate all valid X", "word search in grid", "find a path that satisfies …".

**Reach for this pattern when:**
- You must **enumerate** every solution (not just one, not just count).
- The search forms a **tree of choices**, each path possibly a solution.
- Constraints let you **prune** invalid branches early — that's what makes it tractable.
- Constraints are small (n ≤ ~20, often ≤ 10).

**Don't reach for it when:**
- You only need the **count** or **the best** — DP/greedy is usually faster.
- The branching factor × depth × constraints can't be tamed by pruning — exponential will time out.

## The mental model
DFS with explicit **make-choice / recurse / un-make-choice**. The recursion tree's leaves are candidate solutions. At each node, pick a choice, recurse, then **undo** so the same path slot can try the next choice. Pruning happens by checking constraints *before* recursing.

## Skeleton

```python
# Generic backtracking template
def backtrack(path, choices, result):
    if is_solution(path):
        result.append(path[:])           # snapshot — list is mutated
        return
    for c in choices:
        if not is_valid(path, c): continue
        path.append(c)                   # make choice
        backtrack(path, next_choices(path, c), result)
        path.pop()                       # undo choice

# Subsets — include/exclude
def subsets(nums):
    res, path = [], []
    def go(i):
        if i == len(nums):
            res.append(path[:]); return
        path.append(nums[i]); go(i + 1); path.pop()   # take
        go(i + 1)                                      # skip
    go(0)
    return res

# Permutations — used[] set
def permute(nums):
    res, path, used = [], [], [False] * len(nums)
    def go():
        if len(path) == len(nums):
            res.append(path[:]); return
        for i in range(len(nums)):
            if used[i]: continue
            used[i] = True;  path.append(nums[i])
            go()
            path.pop();      used[i] = False
    go()
    return res
```

## Complexity
- Time: **O(branches^depth)** — e.g., O(n · n!) for permutations, O(2^n · n) for subsets.
- Space: **O(depth)** recursion + O(output) for results.

## Variants in this pattern
1. **Subsets / power set** — include-or-skip at each index.
2. **Permutations** — `used[]` flag or in-place swap to avoid repeats.
3. **Combinations** — start index, pick k of n.
4. **Grid search** — DFS with visited cell marking + un-marking.
5. **Constraint satisfaction** — N-queens, sudoku; prune by row/col/diag/box sets.
6. **String construction** — generate valid parentheses, restore IPs.

## Common bugs / pitfalls
- **Appending `path` instead of `path[:]`** — same list reference; all results end up identical (usually empty by the end).
- **Forgetting to undo** the choice — `path.pop()` / `used[i] = False`. Without it, the sibling branches see stale state.
- **Duplicate results with duplicate input** — sort first, then skip `nums[i] == nums[i-1]` at the same recursion level.
- **Mutating the grid as visited** without restoring — fine if you only need one path, broken if multiple branches share the grid.
- **Wrong base case** — recursing past the last index, indexing OOB.
- **Pruning too late** — check `is_valid(path, c)` *before* appending, not inside the recursive call.

## In 30 seconds
DFS over a choice tree. Make choice → recurse → undo choice. Snapshot solutions with `path[:]`. Prune early or watch time explode.
