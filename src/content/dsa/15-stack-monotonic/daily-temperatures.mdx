## Problem
For each day, find how many days you must wait until a warmer temperature. Return `0` if no warmer day comes.

## Intuition
This is **next greater element** with the answer recorded as a **distance** instead of a value. Sweep left to right with a stack of indices whose answers are still unknown; pop whenever the current temperature exceeds the top, and the answer for the popped index is `i − popped_index`. **Invariant**: stacked indices have strictly decreasing temperatures, and each holds the position of a day still waiting for warmth. Every index is pushed and popped once → O(n).

## Algorithm
Monotone decreasing stack of indices; emit `i − stack.pop()` on each pop.

```python
def daily_temperatures(t):
    n = len(t)
    ans = [0] * n
    stack = []                           # indices, t[stack] strictly decreasing
    for i, v in enumerate(t):
        while stack and t[stack[-1]] < v:
            j = stack.pop()
            ans[j] = i - j
        stack.append(i)
    return ans
```

## Walkthrough
The widget runs on temperatures `[73, 74, 75, 71, 69, 72, 76, 73]`:

1. **i = 0, 1, 2.** Push `0`. At `i=1` (74 > 73): pop `0` → `res[0] = 1`; push `1`. At `i=2` (75 > 74): pop `1` → `res[1] = 1`; push `2`. After three days the stack is `[2]`.
2. **A cooling streak builds the stack.** `i = 3` (71 < 75) and `i = 4` (69 < 71) both push immediately — stack becomes `[2, 3, 4]` (indices of strictly decreasing temps 75, 71, 69).
3. **i = 5 (72) drains two days.** 72 > 69 → pop `4` → `res[4] = 1`. 72 > 71 → pop `3` → `res[3] = 2`. 72 < 75 stops the pops; push `5`. Stack = `[2, 5]`.
4. **i = 6 (76) drains the rest.** 76 > 72 → pop `5` → `res[5] = 1`. 76 > 75 → pop `2` → `res[2] = 4` (four days waited for warmth). Push `6`.
5. **End — i = 7 (73 < 76)** pushes onto the stack. Loop ends with indices 6 and 7 still waiting — their `res` entries stay 0. Final `res = [1, 1, 4, 2, 1, 1, 0, 0]`.
