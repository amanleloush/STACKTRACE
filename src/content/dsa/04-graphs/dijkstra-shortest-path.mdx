## The greedy invariant

At every step, the node with the smallest tentative distance is **finalized** — its distance can't get smaller because every other path would go through a strictly-longer prefix (non-negative edges guarantee this).

So the algorithm is just: extract the min, relax all its outgoing edges, mark it done, repeat.

## Reference implementation

```ts
function dijkstra(adj: Array<Array<[number, number]>>, src: number): number[] {
  const n = adj.length;
  const dist = Array(n).fill(Infinity);
  dist[src] = 0;
  const pq: Array<[number, number]> = [[0, src]];   // [dist, node]
  while (pq.length) {
    pq.sort((a, b) => a[0] - b[0]);                 // toy PQ; use a real heap in prod
    const [d, u] = pq.shift()!;
    if (d > dist[u]) continue;                      // stale entry
    for (const [v, w] of adj[u]) {
      const alt = d + w;
      if (alt < dist[v]) {
        dist[v] = alt;
        pq.push([alt, v]);
      }
    }
  }
  return dist;
}
```

## When *not* to use Dijkstra

- **Negative edges** → Bellman-Ford. O(V·E), slower, but handles negatives. Detects negative cycles too.
- **All-pairs shortest paths** on a dense graph → Floyd-Warshall. O(V³).
- **Unweighted** → BFS. Same shape, O(V + E), no PQ overhead.
- **Hierarchical / road networks** → contraction hierarchies or A* with a good heuristic. Dijkstra explores too much for a known target.

## Why the priority queue choice matters

| PQ implementation | extract-min | decrease-key | total |
|---|---|---|---|
| Sorted array | O(n) | O(n) | **O(V²)** |
| Binary heap | O(log V) | O(log V) | **O((V+E) log V)** |
| Fibonacci heap | O(log V) | O(1) amortized | **O(E + V log V)** |

Binary heap is the practical pick everywhere outside textbooks. The Fibonacci-heap win is asymptotic but the constant factor and code complexity make it rarely worth it in production.
