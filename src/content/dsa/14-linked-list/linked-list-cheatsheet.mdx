## Trigger

**You see in the problem:**
- "Reverse / reorder / partition" a linked list.
- "Detect cycle", "find middle", "Nth from end", "merge".
- "Design LRU / LFU cache" — DLL + hashmap is the canonical structure.
- Constraints say "in-place, O(1) extra space" — the array solution is off-limits.

**Reach for this pattern when:** input is a singly or doubly linked list and you're doing pointer surgery, not value manipulation.

**Don't reach for it when:** you have an array — array is faster everywhere except O(1) insert/delete in the middle (which usually doesn't justify a linked list).

## The mental model

Three habits make linked-list problems easy:

1. **Dummy head** for any operation that might change the head — removes a special case.
2. **Two pointers** at different speeds (or with an offset) for cycle detection, middle, Nth-from-end.
3. **Three pointers** (`prev / curr / next`) for any reverse / reorder / pointer rewrite.

The hardest bug is forgetting to save `curr.next` before overwriting it. Always save first, rewrite second.

## Skeleton — reverse a linked list

```python
def reverse(head):
    prev, curr = None, head
    while curr:
        nxt = curr.next       # save first
        curr.next = prev      # rewrite second
        prev = curr           # advance
        curr = nxt
    return prev
```

## Skeleton — fast/slow pointers

```python
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast: return True
    return False
```

## Skeleton — Nth from end (with dummy head)

```python
def remove_nth_from_end(head, n):
    dummy = ListNode(0, head)
    slow = fast = dummy
    for _ in range(n + 1):
        fast = fast.next            # build a gap of n+1 (slow to fast.next)
    while fast:
        slow, fast = slow.next, fast.next
    slow.next = slow.next.next
    return dummy.next
```

## Complexity

- Time: O(n) for every standard linked-list traversal.
- Space: O(1) extra — that's why we use these patterns at all.
- Recursion-based reversal uses O(n) stack space — interviewers sometimes ask for the iterative version explicitly.

## Variants in this pattern

1. **Reverse / reverse-in-range** — three-pointer rewrite, watch the boundaries.
2. **Reverse in K-groups** — reverse blocks of K, glue them; pre-check that K nodes remain.
3. **Cycle detection (Floyd)** — slow/fast; second phase to find cycle start (slow restarts at head, both move 1).
4. **Find middle** — slow/fast; on even length, "middle" can be either floor or ceiling — clarify with the interviewer.
5. **Merge two / K sorted** — dummy head + tail; for K, use a min-heap of heads.
6. **Reorder list** — find middle → reverse second half → weave.
7. **LRU / LFU cache** — DLL + hashmap; map key → DLL node for O(1) move-to-head.
