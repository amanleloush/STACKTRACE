## Trigger
**You see in the problem:** "sorted array + pair/triplet with sum X", "in-place remove/dedupe", "palindrome check", "two walls / heights → area or volume", "partition by predicate".

**Reach for this pattern when:**
- Input is **sorted** (or you can sort it) and the question is about pairs/triplets.
- You must operate **in place** with O(1) extra space.
- A **symmetric** property — palindrome, reverse, equality from both ends.
- Two boundary values define an **area / capacity** (Container, Trapping Rain).

**Don't reach for it when:**
- Order matters and you can't sort (would destroy the answer's index).
- You need every contiguous subarray's stat — that's sliding window.
- The pair condition isn't monotone in pointer movement (no clean "move which side" rule).

## The mental model
Two indices walk the array under an **invariant**: everything outside `[l, r]` (converging) or before `slow` (partitioning) is already resolved. Each step inspects the two values, makes **one decision** — advance `l`, advance `r`, or write+advance — and the search space shrinks by 1. O(n) total because each pointer moves monotonically.

## Skeleton

```python
# Converging pointers (sorted array, find pair)
def two_sum_sorted(arr, target):
    l, r = 0, len(arr) - 1
    while l < r:
        s = arr[l] + arr[r]
        if s == target: return [l, r]
        elif s < target: l += 1     # need bigger -> move left side up
        else:            r -= 1     # need smaller -> move right side down
    return [-1, -1]

# Slow/fast partition (in-place dedupe / move zeros)
def remove_dups(arr):
    slow = 0
    for fast in range(len(arr)):
        if fast == 0 or arr[fast] != arr[slow - 1]:
            arr[slow] = arr[fast]
            slow += 1
    return slow  # length of unique prefix
```

## Complexity
- Time: **O(n)** (or O(n log n) if you have to sort first)
- Space: **O(1)**

## Variants in this pattern
1. **Converging from ends** — sorted array + pair sum. `l++`/`r--` based on compare.
2. **Slow/fast partition** — in-place write head + scan head.
3. **Same-direction (sliding-like)** — both start at 0, fast races ahead.
4. **Triplet sweep (3Sum)** — fix one element, two-pointer the rest. O(n²).
5. **Two-pass meet-in-middle** — precompute max-from-left + max-from-right (trapping rain water).
6. **Dutch national flag** — three pointers, partition into &lt;pivot / =pivot / &gt;pivot.

## Common bugs / pitfalls
- **Off-by-one on the while condition**: `l < r` vs `l <= r`. For pair-search use `<` (pair needs two distinct indices); for binary-style scans `<=`.
- **3Sum duplicates**: after a match, advance `l` *and* `r` past equal values; also skip duplicate `i` at the outer loop.
- **Moving the wrong pointer** in Container — you must move the *shorter* side; moving the taller can never improve area.
- **Forgetting to sort** when the problem allows it — without sort, converging logic breaks.
- **Slow/fast off-by-one**: `slow` is the write index; final answer length is `slow`, not `slow - 1`.
