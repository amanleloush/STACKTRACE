## Trigger
**You see in the problem:** "longest/shortest **contiguous** subarray/substring with property X", "fixed window of size k", "all anagrams/permutations in S", "max/min in every window of size k".

**Reach for this pattern when:**
- Subarray/substring must be **contiguous** (not just any subset).
- Property is **monotone-ish** — extending the window can only make it "more broken", shrinking can only help.
- You'd otherwise need O(n²) over all subarrays — window collapses it to O(n).

**Don't reach for it when:**
- Items can be reordered or non-contiguous picks are allowed → DP / two-pointer-on-sorted.
- Property isn't monotone (adding an element could both fix and break) — usually needs prefix sums + hash, or DP.

## The mental model
Two indices `l ≤ r` define a window `[l, r]`. **Expand** by moving `r` until the window violates the constraint; **contract** by moving `l` until it's valid again. Each index moves at most n times → **O(n)** amortized. The window carries a tiny "state" (sum, count, hash map of freqs) you update in O(1) per step.

## Skeleton

```python
# Variable-size window: longest substring with at most K distinct chars
def longest_at_most_k_distinct(s, k):
    from collections import defaultdict
    freq = defaultdict(int)
    l = 0
    best = 0
    for r, ch in enumerate(s):
        freq[ch] += 1
        while len(freq) > k:           # contract until valid
            freq[s[l]] -= 1
            if freq[s[l]] == 0: del freq[s[l]]
            l += 1
        best = max(best, r - l + 1)    # window now valid
    return best

# Fixed-size window: max sum of window length k
def max_sum_k(arr, k):
    s = sum(arr[:k])
    best = s
    for r in range(k, len(arr)):
        s += arr[r] - arr[r - k]       # slide: add new, drop old
        best = max(best, s)
    return best
```

## Complexity
- Time: **O(n)** — each index moves forward at most n times.
- Space: **O(k)** or **O(Σ)** for the freq map (alphabet size or distinct count).

## Variants in this pattern
1. **Fixed-size window** — `r - l + 1 == k`; slide by 1, recompute in O(1).
2. **Variable-size, longest valid** — expand always, shrink while invalid; track max length.
3. **Variable-size, shortest valid** — expand until valid, shrink while *still* valid; track min length.
4. **Exactly K** — `atMost(K) - atMost(K-1)` trick (subarrays with exactly K distinct).
5. **Monotonic-deque window** — max/min in sliding window in O(n) using a deque.
6. **String anagram window** — fixed-size with freq-map equality check.

## Common bugs / pitfalls
- **Returning the wrong window**: for "shortest valid", capture best *inside* the shrink loop; for "longest valid", capture *after* shrinking.
- **Forgetting to delete zero-count keys** in the freq map — `len(freq)` becomes wrong for "distinct chars" problems.
- **Window length off-by-one**: it's `r - l + 1`, not `r - l`.
- **Fixed-window first fill**: don't forget to seed the first `k` elements before sliding.
- **Negative numbers** break the "sum-based" monotone assumption — sliding window for sum ≥ target only works with non-negatives.
